Dr.HMF — AI Medical Agent

classification · ML
// problem

People describing symptoms in plain language have no fast way to get a shortlist of likely conditions before seeing a doctor — most tools either need exact medical terms or give one flat answer with no confidence signal.

// approach

Framed as a multi-class classification task: given a symptom description, predict the condition, evaluated with macro F1 (not plain accuracy) since common conditions vastly outnumber rare ones in training data, and top-3 accuracy as a realistic, safe bar for a decision-support tool.

// what I built

An end-to-end assistant that takes structured or free-text symptom input and returns a confidence-ranked shortlist of likely conditions, with supporting guidance rather than a single unexplained answer.

// result

Correctly frames diagnosis as decision support, not a final answer — the top-3 shortlist format matches how a person would actually use it alongside a real doctor.

stack: Python · ML classification pipeline · web interface
